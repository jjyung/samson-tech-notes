# 標題

一句 hook：

「GraphQL 很酷，但大型企業架構師常常拒絕它。」

---

## 問題

很多人以為 GraphQL 是 API 的未來。

但在大型企業，情況往往相反。

---

## 為什麼企業架構師會抗拒 GraphQL

### 1 可觀測性

GraphQL query dynamic

難以做 caching / monitoring

---

### 2 安全性

overfetch

query depth attack

---

### 3 團隊成本

學習曲線

---

## 但 AI Agent 時代可能改變

Agent 可以：

- 讀 schema
- 自動生成 query

GraphQL suddenly makes sense.

---

## 結論

GraphQL 不是壞技術

只是很多團隊用錯地方。
