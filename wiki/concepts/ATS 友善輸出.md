# ATS 友善輸出

## 核心想法

履歷的最後輸出不只是美觀問題，也涉及 ATS 能不能正確解析。對大多數場景來說，先追求可讀、可解析、格式穩定，比花俏排版更重要。

## 典型原則

- 使用標準 section 標題
- 盡量避免多欄、複雜表格、過多圖示
- 保持 bullet 簡潔
- 日期格式一致
- 對機器可讀，也對人可讀

## 為什麼重要

即使內容正確，如果格式不利於 ATS 解析，實際投遞效果也會下降。輸出層應該服務於後續分發與掃描，而不是只服務視覺展示。

## 實作建議

- 先輸出 Markdown 或簡單結構化文本
- 再由 builder 轉成 PDF / DOCX / HTML
- 把 ATS 檢查納入 QA gate
- 若不同角色有不同慣例，再做 role-specific template
