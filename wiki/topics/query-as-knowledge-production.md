# Query 即知識生產

## 核心想法

在一般聊天流程裡，提問的答案常常只存在於對話視窗中，之後就消失了。這種模式浪費了很多高價值的綜合整理。

這篇 `LLM Wiki` 提出的觀點是：好的 query 結果本身就是知識，應該被寫回 wiki。

## 哪些答案值得回寫

- 比較分析
- 決策備忘錄
- 主題綜述
- 概念澄清
- 多篇來源交叉整理後得到的結論

## 為什麼這很重要

- 同樣問題下次不用再做一次
- 綜合思考可以累積，而不是只留在聊天紀錄裡
- 查詢結果可以反過來強化 wiki 的主題結構
- 問題本身會推動知識庫長出新的頁面

## 實務上怎麼做

當一個回答已經超過「臨時說明」的程度，就可以考慮把它整理到：

- `wiki/queries/`：偏一次性分析、比較、整理
- `wiki/topics/`：已經上升成長期主題頁
- `wiki/concepts/`：已經形成可重複引用的概念頁

## 跟 Ingest 的差別

- ingest 是從原始來源出發
- query 是從問題與綜合思考出發

兩者最後都會回到同一件事：讓 wiki 變得更完整。

## 對這個 Repo 的啟發

這代表你不需要把 wiki 想成「讀完資料後才更新」。很多真正有價值的知識，其實是在提問、比較、重組之後才被產生出來，而那些內容也值得正式留存。
