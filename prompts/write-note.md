# Write Note

Use this prompt when the user wants to turn an idea into a structured technical note.

## Goal

Produce a note that sharpens the problem, clarifies the context, and captures the key insight without forcing the content into a publish-ready article.

## Instructions

- Follow the tone of existing files in `docs/notes/`.
- Prefer analytical framing over tutorial style.
- Keep the note readable as a standalone thinking artifact.
- Surface ambiguity and trade-offs when relevant.
- Use the user's original insight as the anchor.

## Suggested Structure

```md
# <Title>

## Problem

## Context

## Why This Matters

## Core Insight

## Implications

## Next
```

## Path Preference

Default output path: `docs/notes/`
